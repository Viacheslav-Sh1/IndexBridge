# IndexBridge

Универсальный агрегатор торрент-индексаторов и промежуточный слой для Jackett, Prowlarr и Jacred.

Объединяет результаты из **Jackett**, **Prowlarr** и **Jacred**, предоставляя единый API с фильтрацией, дедупликацией и нормализацией метаданных.

> [!NOTE]
> IndexBridge не заменяет Jackett, Prowlarr или Jacred.
>
> Он выступает промежуточным слоем, объединяя несколько источников, выполняя фильтрацию, дедупликацию и нормализацию результатов перед их выдачей через единый API.

```text
Jackett ─┐
Prowlarr ├──► IndexBridge ───► Клиент
Jacred ──┘
```

## Почему IndexBridge?

Если у вас несколько индексаторов, они могут возвращать результаты в разных форматах, дублировать раздачи и выдавать несогласованные метаданные. IndexBridge нормализует всё, убирает дубликаты и отдаёт чистые отфильтрованные результаты одним запросом в формате Jacred.

Кроме того, вместо того чтобы публиковать несколько сервисов наружу, можно оставить их доступными только внутри локальной сети и предоставить внешний доступ через один экземпляр IndexBridge.
```text
                Интернет
                    │
                    ▼
          ┌─────────────────┐
          │   IndexBridge   │
          │ (один API-ключ) │
          └─────────────────┘
                    │
                    ▼
          ┌─────────────────┐
          │  Локальная сеть │
          ├─────────────────┤
          │ Jackett #1      │
          │ Jackett #2      │
          │ Prowlarr #1     │
          │ Prowlarr #2     │
          │ Jacred #1       │
          │ Jacred #2       │
          └─────────────────┘
```

**Что это даёт:**

- **Один открытый порт** вместо нескольких
- **Один внешний API-ключ** — ключи Jackett, Prowlarr и Jacred никогда не покидают локальную сеть
- **Скрытая инфраструктура** — внутренние адреса и ключи не видны внешним клиентам
- **Централизованная фильтрация** — единые правила для всех источников
- **Дедупликация между провайдерами** — исключение дублей до того как они дойдут до клиента
- **Замена провайдеров без перенастройки клиентов** — можно добавить или удалить источник, внешний URL остаётся прежним

**Для клиента IndexBridge выглядит как обычный Jackett-совместимый сервер:**

**Производительность:** все провайдеры опрашиваются параллельно. Время ответа обычно определяется самым медленным успешно ответившим провайдером и настройкой `PROVIDER_TIMEOUT`.

## Примеры работы

### Jackett через IndexBridge

Один и тот же клиент (Lampa), одни и те же торренты и один и тот же источник.
IndexBridge дополняет результаты Jackett метаданными, извлечёнными из названий раздач, и возвращает их в формате Jacred.

Появляются:

- языки аудиодорожек;
- языки субтитров;
- разрешение видео;
- информация об аудио;
- дополнительные метаданные для клиента.

IndexBridge не заменяет Jackett и не изменяет сами раздачи. Он лишь дополняет результаты метаданными, извлечёнными из названия релиза.

### Общая выдача из нескольких источников

IndexBridge может одновременно агрегировать Jackett, Prowlarr и Jacred.

- результаты объединяются;
- выполняется фильтрация;
- большинство дубликатов удаляется автоматически;
- метаданные Jacred сохраняются без изменений;
- для результатов Jackett и Prowlarr недостающие данные извлекаются из заголовков.  


![Общая выдача из нескольких источников](docs/images/multi-provider.webp)


> Дедупликация не гарантирует полное удаление всех дубликатов. Если разные источники возвращают несовместимые или неполные идентификаторы, одинаковые раздачи могут присутствовать в итоговой выдаче.
>
> Это осознанный компромисс: лучше получить несколько рабочих вариантов одного релиза, чем случайно потерять единственную доступную раздачу.

## Возможности

- **Агрегация из нескольких источников** — Jackett, Prowlarr, Jacred (без ограничения количества)
- **Параллельные запросы** — все провайдеры опрашиваются одновременно через `asyncio.gather()`
- **Настраиваемый таймаут** — медленные провайдеры отваливаются по таймауту, быстрые продолжают работать
- **Circuit breaker** — после нескольких последовательных ошибок провайдер временно исключается из опроса
- **RSS-кеш** — пустые запросы (без поискового запроса) кешируются, чтобы не дёргать провайдеров каждые 30 секунд
- **Приоритет провайдеров** — при дубликате остаётся результат от провайдера с наивысшим приоритетом
- **Нет каскадных сбоев** — один упавший провайдер не влияет на остальные
- **Гибкий доступ к провайдерам** — можно получить как объединённый результат (`/all`), так и результат от конкретного провайдера (`/имя`)
- **Парсинг заголовков** — для Jackett и Prowlarr метаданные извлекаются из названия раздачи (языки, озвучки, качество, кодеки, сезоны, серии)
- **Автоопределение формата** — если источник уже возвращает данные в формате Jacred (поля `ffprobe`, `languages`, `info`), метаданные используются как есть, без повторного парсинга
- **Дедупликация** — по BTIH-хешу, ссылке на раздачу или названию+размеру
- **Расширенная фильтрация** — сидеры, пиры, качество, языки, озвучки, ключевые слова
- **Объединённый эндпоинт** — `/all` возвращает дедуплицированные результаты со всех источников
- **Выдача в формате Jacred** — все результаты отдаются в формате Jacred (с полями `ffprobe`, `languages`, `info`)
- **Ограничение запросов** — изоляция по IP с автоматической очисткой
- **Безопасность** — API-ключ, скрытые служебные эндпоинты, защитные заголовки
- **Docker** — один контейнер, простая установка

## Архитектура
```text
Jackett #1 ─┐
Jackett #2 ─┤
Prowlarr ───┼──► IndexBridge ──► /имя
Jacred #1 ──┤                 └► /all
Jacred #2 ──┘
```
Все провайдеры опрашиваются параллельно, результаты объединяются, фильтруются и дедуплицируются.

### Обработка источников

- Jackett / Prowlarr — метаданные извлекаются из заголовков.
- Jacred — готовые метаданные используются без повторного парсинга.

## Дедупликация

Результаты от всех провайдеров объединяются и проверяются на дубликаты по трём уровням (в порядке приоритета):

1. **BTIH-хеш** — из magnet-ссылки извлекается хеш торрента. Самый надёжный способ.
2. **Details** — ссылка на страницу раздачи. Используется если нет magnet-ссылки.
3. **Title + Size** — запасной вариант, если нет ни хеша, ни ссылки.

Если найден дубликат, выбор происходит по одному из двух правил:

- **С приоритетом провайдеров** — если задан `PROVIDER_PRIORITY`, остаётся результат от провайдера с наивысшим приоритетом (первый в списке)
- **Без приоритета** — остаётся раздача с наибольшим количеством сидеров (поведение по умолчанию)

### Ограничения дедупликации

Если разные провайдеры возвращают неполные или несовместимые идентификаторы, некоторые одинаковые раздачи могут остаться в выдаче.
Это ожидаемое поведение. Предпочтение отдаётся сохранению доступных релизов, а не агрессивному удалению потенциальных дубликатов.
Если дубликатов слишком много, можно:
- отключить `PROVIDER_PRIORITY`;
- использовать только один источник для конкретного трекера;
- создать Issue с примерами.

## Отказоустойчивость и производительность

### RSS-кеш

Пустые запросы (без параметра `Query`) кешируются на заданное время. 
Это стандартное поведение для Jackett/Prowlarr — RSS-лента обновлений, которую клиенты опрашивают каждые 30 секунд. 
Без кеша IndexBridge при каждом таком запросе дёргал бы всех провайдеров.
```env
RSS_CACHE_TTL=300  # время жизни кеша в секундах (по умолчанию 300 = 5 минут)
```
В течение этого времени все пустые запросы получают закешированный ответ. Поисковые запросы (с Query) не кешируются.


### Circuit breaker
После нескольких последовательных ошибок или таймаутов провайдер временно отключается. 
Это защищает от зависаний: проблемный провайдер не замедляет запросы к остальным.

```env
CB_MAX_FAILS=3      # количество ошибок подряд для отключения (по умолчанию 3)
CB_TIMEOUT=300      # время отключения в секундах (по умолчанию 300 = 5 минут)
```
Как работает:
Провайдер отвечает с ошибкой или не укладывается в PROVIDER_TIMEOUT
После CB_MAX_FAILS ошибок подряд провайдер отключается на CB_TIMEOUT секунд

В логах появляется запись:
WARNING: [jac] circuit breaker OPEN for 300s
По истечении времени провайдер снова опрашивается

### Приоритет провайдеров
Порядок доверия к источникам при дедупликации. 
Первый в списке имеет наивысший приоритет.

```env
PROVIDER_PRIORITY=jac,jred,pr  # jac > jred > pr
```

При дубликате остаётся результат от провайдера с наивысшим приоритетом. 
Если приоритет не задан — по количеству сидеров.

Пример:
Одна и та же раздача найдена в двух провайдерах:

| Провайдер	| Сидеров |
|-----|----|
| jred | 25 |
| jac | 5 |

Без приоритета: останется jred (25 сидеров > 5)

С приоритетом jac,jred: останется jac (он первый в списке, ему доверяем больше)

Приоритет полезен когда:
- Хочется отдавать предпочтение проверенному источнику
- Jacred передаёт неактуальное количество сидеров — можно поставить его ниже 
  в списке, чтобы при дубликате оставались свежие данные от Jackett/Prowlarr

**Сколько провайдеров можно подключить?**
Технически: В NAMES нет лимита.

Характеристики производительности:

| Провайдеров | Статус | Рекомендация |
|-------------|---------|---------------|
| 1-5 | ✅ Идеально | Настроек по умолчанию достаточно |
| 5-15 | ✅ Хорошо | Увеличить `PROVIDER_TIMEOUT` до 20-30 с |
| 15-30 | ⚠️ Возможны задержки | Увеличить таймаут (не тестировалось) |
| 30+ | ⚠️ Требуется тестирование | Может не хватить CPU/памяти (не тестировалось) |

Время ответа примерно равно времени ответа самого медленного провайдера плюс накладные расходы на обработку (~100-200мс). 
Время ответа не равно сумме времени ответа всех источников.

Пример с 5 провайдерами (каждый ~1с):  
Последовательно: ~5 секунд  
IndexBridge: ~1 секунда  

## Быстрый старт

### 1. Скопируйте шаблон конфигурации
```bash
cp .env.example .env
```
### 2. Отредактируйте настройки
```bash
nano .env
```
**Важно**:
- Замените все значения на свои
- Удалите строки провайдеров, которых у вас нет
- Имена провайдеров в NAMES должны совпадать с именами в переменных `JACKETT_{name}_URL / PROWLARR_{name}_URL`
- Если в NAMES указан провайдер, для которого нет URL и API_KEY, IndexBridge запустится, но этот провайдер будет пропущен.

`PROXY_API_KEY=ваш-секретный-ключ`  
`NAMES=jred,jac,pr`  
`JACKETT_jred_URL=https://remote.example.com`  
`JACKETT_jred_API_KEY=ключ-jred`  
`JACKETT_jac_URL=http://jackett:9117`  
`JACKETT_jac_API_KEY=abc123`  
`PROWLARR_pr_URL=http://prowlarr:9696`  
`PROWLARR_pr_API_KEY=def456`  

### 3. Соберите и запустите контейнер
**Вариант 1 — Docker**
```bash
docker build -t indexbridge .
docker run -d --name indexbridge --env-file .env -p 8000:8000 indexbridge
```
**Вариант 2 — Docker Compose**
```bash
docker-compose up -d --build
```
### docker-compose.yml
```yaml
version: '3.8'

services:
  indexbridge:
    build: .
    image: indexbridge:latest
    container_name: indexbridge
    restart: unless-stopped
    ports:
      - "8000:8000"
    environment:
      - LOG_LEVEL=INFO
      - PROXY_API_KEY=${PROXY_API_KEY}
      - TZ=Europe/Kyiv
      - PORT=8000
      - COMBINED_NAME=all
      - NAMES=${NAMES}
      # Укажите своих провайдеров, удалите неиспользуемые
      # - PROVIDER_PRIORITY=jac,pr
      - JACKETT_jac_URL=${JACKETT_jac_URL}
      - JACKETT_jac_API_KEY=${JACKETT_jac_API_KEY}
      - PROWLARR_pr_URL=${PROWLARR_pr_URL}
      - PROWLARR_pr_API_KEY=${PROWLARR_pr_API_KEY}
      - RATE_LIMIT_ENABLED=true
      - RATE_LIMIT_REQUESTS=60
      - RATE_LIMIT_WINDOW=60
      - FILTER_MIN_SEEDERS=1
      - PROVIDER_TIMEOUT=15
      - RSS_CACHE_TTL=300
      - CB_MAX_FAILS=3
      - CB_TIMEOUT=300
```
**Важно**: удалите из environment строки провайдеров, которые вы не используете. Для каждого провайдера из NAMES должны быть заданы `JACKETT_{name}_URL` и `JACKETT_{name}_API_KEY` (или `PROWLARR_{name}_URL` и `PROWLARR_{name}_API_KEY`).

### 4. Проверка

**Отдельный провайдер**
```bash
curl "http://localhost:8000/jac?apikey=ваш-секретный-ключ&Query=Foundation"
```

**Все провайдеры вместе**
```bash
curl "http://localhost:8000/all?apikey=ваш-секретный-ключ&Query=Foundation"
```

**Torznab-совместимый путь (без имени провайдера)**
```bash
curl "http://localhost:8000/api/v2.0/indexers/all/results?apikey=ваш-секретный-ключ&Query=Foundation"
```
API эндпоинты:

| Эндпоинт | Описание |
|------------|--------------|
| /{имя} | Результаты одного провайдера |
| /{имя}/api/v2.0/indexers/all/results | Jackett-совместимый путь |
| /api/v2.0/indexers/all/results | Torznab без имени (объединённый) |
| /{COMBINED_NAME} | Объединённые результаты со всех провайдеров (по умолчанию /all) |

## Авторизация

Все запросы требуют API-ключ:
**Query-параметр**
?apikey=ваш-ключ

**Заголовок**
X-API-Key: ваш-ключ
Неверный или отсутствующий ключ возвращает 404 Not Found.

### Настройка провайдеров

**Jackett (в том числе Jacred как Jackett-совместимый сервер)**  
`JACKETT_{имя}_URL=http://хост:9117`  
`JACKETT_{имя}_API_KEY=ключ`  

**Prowlarr**  
`PROWLARR_{имя}_URL=http://хост:9696`  
`PROWLARR_{имя}_API_KEY=ключ`  

**Важно**: Jacred подключается как обычный Jackett-провайдер.  
IndexBridge автоматически определит, что ответ уже в Jacred-формате, и не будет парсить заголовки повторно.

Можно добавить сколько угодно провайдеров. Все опрашиваются параллельно.

Пример с 6 провайдерами:

```env
NAMES=jac1,jac2,jac3,pr1,pr2,jred1

JACKETT_jac1_URL=http://10.0.0.1:9117
JACKETT_jac1_API_KEY=ключ-jac1

JACKETT_jac2_URL=http://10.0.0.2:9117
JACKETT_jac2_API_KEY=ключ-jac2

JACKETT_jac3_URL=http://10.0.0.3:9117
JACKETT_jac3_API_KEY=ключ-jac3

PROWLARR_pr1_URL=http://10.0.0.3:9696
PROWLARR_pr1_API_KEY=ключ-pr1

PROWLARR_pr2_URL=http://10.0.0.4:9696
PROWLARR_pr2_API_KEY=ключ-pr2

JACKETT_jred1_URL=https://remote.example.com
JACKETT_jred1_API_KEY=ключ-jred1
```

### Производительность

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| PROVIDER_TIMEOUT | 15 | Максимальное время ожидания провайдера (сек) |
| RSS_CACHE_TTL | 300 | Время жизни кеша пустых запросов (сек) |
| CB_MAX_FAILS | 3 | Ошибок подряд до отключения провайдера |
| CB_TIMEOUT | 300 | Время отключения провайдера (сек) |
| PROVIDER_PRIORITY | — | Приоритет провайдеров при дедупликации: jac,jred,pr |

Как изменить:
В .env:
```env
PROVIDER_TIMEOUT=30
RSS_CACHE_TTL=600
CB_MAX_FAILS=5
CB_TIMEOUT=600
PROVIDER_PRIORITY=jac,jred,pr
```
Или в docker-compose.yml:
```yml
environment:
  - PROVIDER_TIMEOUT=30
  - RSS_CACHE_TTL=600
  - CB_MAX_FAILS=5
  - CB_TIMEOUT=600
  - PROVIDER_PRIORITY=jac,jred,pr
 ``` 
После изменения перезапустите контейнер.

**Рекомендации:**
Локальные провайдеры: PROVIDER_TIMEOUT=10-15, удалённые: 15-30  
RSS_CACHE_TTL не меньше интервала опроса RSS-клиента (обычно 300)  
CB_MAX_FAILS=3 и CB_TIMEOUT=300 — хорошие значения по умолчанию  
PROVIDER_PRIORITY — первым указывайте наиболее доверенный источник  
Уменьшите PROVIDER_TIMEOUT, если предпочитаете быстрые частичные результаты вместо ожидания всех  

### Дополнительные параметры

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| COMBINED_NAME | all | Имя объединённого эндпоинта (например, combined → /combined) |

### Фильтрация

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| FILTER_MIN_SEEDERS | 1 | Минимальное количество сидеров |
| FILTER_MAX_SEEDERS | 0 | Максимальное количество сидеров (0=безлимит) |
| FILTER_MIN_PEERS | 0 | Минимальное количество пиров |
| FILTER_LANGUAGES | — | Запрещённые языки: rus,ukr,eng (пусто = все разрешены) |
| FILTER_VOICES | — | Запрещённые студии озвучки (пусто = все разрешены) |
| FILTER_BLACKLIST_TITLES | — | Запрещённые слова в заголовке |
| FILTER_QUALITY_MIN | 0 | Минимальное качество |

### Ограничение запросов

### Типовые сценарии
- несколько Jackett → один IndexBridge → клиент;  
- несколько Prowlarr → один IndexBridge → клиент;  
- несколько Jacred → один IndexBridge → клиент;  
- смешанные источники через единый API;  
- одна внешняя точка доступа вместо нескольких сервисов;  
- единая фильтрация и дедупликация результатов;  
- единый API поверх нескольких индексаторов.  

> [!IMPORTANT]
> IndexBridge не заменяет Jackett, Prowlarr и Jacred.  
>
> Он не выполняет индексацию и не ищет раздачи самостоятельно.  
>
> IndexBridge является промежуточным слоем между индексаторами и клиентами, обеспечивая агрегацию, фильтрацию, дедупликацию и нормализацию результатов.
